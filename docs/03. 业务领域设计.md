# 业务领域设计

## 1. 会计基础

SmartFlow 在数据层严格使用复式记账法，确保每一笔交易的资金流向都可追溯、可平账、可审计。

### 1.1 会计恒等式

所有数据流转遵循以下三个等式：

- **基础恒等式**：`资产 = 负债 + 所有者权益`
- **动态恒等式**：`资产 = 负债 + 所有者权益 + (收入 - 支出)`
- **记账操作等式**：`资产 + 支出 = 负债 + 所有者权益 + 收入`

记账操作等式将动态恒等式中的"-"消去，使所有变量都为非负数，是底层分录写入的形式基础。

### 1.2 五大账户类型

系统将所有"资金载体"和"收支分类"统一抽象为账户，分为五大类型：

| 账户类型 | 标识 | 业务含义 | 典型示例 |
| :--- | :--- | :--- | :--- |
| 资产 | `ASSET` | 资金的实际载体或应收款 | 现金、储蓄卡、第三方支付、基金、报销应收款 |
| 负债 | `LIABILITY` | 欠款或透支额度 | 信用卡、花呗、房贷、车贷、个人借款 |
| 所有者权益 | `EQUITY` | 净资产的调节项 | 期初余额、平账盈亏 |
| 收入 | `INCOME` | 资金流入的分类 | 工资、兼职、投资收益 |
| 支出 | `EXPENSE` | 资金流出的分类 | 餐饮、交通、房租 |

### 1.3 借贷映射规则

每一笔交易至少涉及两个账户，且**借方金额合计 = 贷方金额合计**。账户金额变化与借贷方向的映射关系如下：

| 账户类型 | 余额增加 | 余额减少 |
| :--- | :--- | :--- |
| 资产 / 支出 | 借（DEBIT） | 贷（CREDIT） |
| 负债 / 收入 / 所有者权益 | 贷（CREDIT） | 借（DEBIT） |

借贷方向是分录（`entries`）的核心字段，账户余额随分录过账自动累计。


## 2. 核心数据模型

SmartFlow 的数据模型围绕“统一账户树 + 交易主记录 + 业务分项 + 复式分录”展开。详细字段、约束与索引设计见《03.1 数据模型》。

| 表 | 定位 |
| :--- | :--- |
| `accounts` | 单表树形结构，统一承载资产、负债、权益、收入、支出五大类账户；收入 / 支出分类也是账户树的一部分。 |
| `transactions` | 交易主记录，承载业务用途、业务关系、业务状态、更正链路、来源以及统计 / 预算排除标记。 |
| `transaction_details` | 业务详情分项，承载 `detail_type` 与分项金额，用于表达还款、报销等复合交易结构；不参与账户余额、收支统计或预算统计。 |
| `entries` | 复式记账分录，承载账户、借贷方向与金额；账户余额、分类聚合、收支统计与预算统计均从 entries 推导。 |
| `budgets` | 月度预算，绑定支出分类账户；总预算使用空分类账户表达。 |

核心取舍：

- 所有“资金载体”和“收入 / 支出分类”统一进入 `accounts`，避免业务分类与账务账户双层映射。
- `transaction_details` 只表达业务分项，不再承载统计口径。
- 统计与预算统一从 `entries` 推导，并通过 `transactions.business_state`、`is_excluded_from_stats`、`is_excluded_from_budget` 过滤。

详细内容见《03.1 数据模型》。

## 3. 核心业务规则

核心业务规则定义“交易如何写入、如何更正、如何删除、如何统计”。完整规则见《03.2 核心业务规则》，各场景的三表示例见《03.3 业务场景流程示例》。

### 3.1 写账规则总览

- 一笔交易在同一数据库事务内写入 1 条 `transactions`、1 条或多条 `transaction_details`、2 条或多条 `entries`。
- `entries` 必须借贷平衡；`transaction_details` 不参与借贷平衡。
- 写入 entries 后同步更新 `accounts.balance_minor`。

### 3.2 更正与删除总览

- 已入账交易不直接覆盖原记录。更正 = 冲销原交易 + 重记正确交易。
- 删除 = 冲销原交易且不补记。
- `mutation_kind` 表达更正链角色：`ORIGINAL / CORRECTION / REVERSAL`。
- `business_state` 表达用户业务视角状态：`CURRENT / REPLACED / CANCELED / COMPENSATION`。默认流水、收支统计和预算统计只读取 `CURRENT`。

### 3.3 统计规则总览

- 账户余额：从 entries 按账户类型和借贷方向累计。
- 收入统计：统计 INCOME 分类账户 entries，CREDIT 为正收入，DEBIT 为负收入。
- 支出统计：统计 EXPENSE 分类账户 entries，DEBIT 为正支出，CREDIT 为负支出。
- 预算统计：统计支出分类 entries，并过滤 `transactions.is_excluded_from_budget = true` 的交易。

### 3.4 业务场景总览

系统当前覆盖以下业务用途：

- 普通支出 / 普通收入
- 普通转账（含手续费）
- 退款
- 报销垫付 / 报销到账 / 结束报销
- 还款（本金、利息、手续费）
- 借入
- 期初余额
- 余额调整

详细规则见《03.2 核心业务规则》；表级示例见《03.3 业务场景流程示例》。
