# 技术实现要点

## 1. 数据一致性保障

**复式记账校验：**
* 每笔交易必须至少包含两条分录（entries）。
* 借方总额必须等于贷方总额。
* 插入交易时需同步更新账户余额。


**余额计算逻辑：**
* 资产/支出账户：借方 +，贷方 -
* 负债/收入/权益账户：贷方 +，借方 -


## 2. 账单导入难点

**字段映射模板库：**
* 不同平台的账单格式差异大，需建立映射模板。
* 支付宝：交易时间、交易对方、商品说明、金额、收/支、交易状态。
* 微信：交易时间、交易类型、交易对方、商品、金额(元)、支付方式、当前状态。
* 银行：交易日期、交易摘要、对方账号、对方户名、交易金额、账户余额。


**重复交易检测：**
* 根据时间 + 金额 + 交易对方进行模糊匹配。
* 提供用户确认界面，避免误判。


## 3. AI 分类实现建议

**Prompt 设计：**
```
你是一个财务分类助手。根据以下信息，判断这笔交易应该归类到哪个分类。

用户的历史分类规则：
- “麦当劳” -> 餐饮 > 快餐
- “滴滴出行” -> 交通 > 打车
- “中国石化” -> 交通 > 加油

当前交易信息：
- 交易对方：星巴克
- 商品说明：美式咖啡
- 金额：35 元

请返回分类路径（如：餐饮 > 咖啡）和置信度（0-1）。
```


**置信度机制：**
* 置信度 > 0.8：自动应用分类。
* 置信度 0.5-0.8：提示用户确认。
* 置信度 < 0.5：要求用户手动选择。


## 4. 性能优化

**数据查询优化：**
* 为 `transactions.tx_time` 建立索引。
* 为 `entries.account_id` 建立索引。


**余额计算优化：**
* 账户余额采用增量更新，避免每次查询时重新计算。
* 定期校验余额准确性（通过分录汇总对比）。
