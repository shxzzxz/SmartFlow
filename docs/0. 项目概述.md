# 项目概述

## 项目定位

SmartFlow 是一款基于**复式记账法**的个人财务管理系统，纯前端实现，数据存储在本地浏览器。

## 核心特点

- **会计原则驱动**：严格遵循复式记账法，确保资产负债表平衡（详见 [核心业务模型](./1.%20核心业务模型.md)）
- **纯前端架构**：无需服务器，数据完全本地化，保护隐私
- **智能化辅助**：支持账单导入、AI 分类、自动化规则（详见 [核心功能](./2.%20核心功能.md)）
- **多维度分析**：提供结构、趋势、对比等多种财务分析视图

## 技术栈

- **前端框架**：Vue 3 + Vite + TypeScript
- **状态管理**：Pinia + Vue Router 4
- **本地数据库**：Dexie.js (IndexedDB)
- **UI 框架**：TailwindCSS + shadcn-vue
- **图表库**：ECharts (vue-echarts)
- **工具库**：currency.js、dayjs、PapaParse

## 核心模块

1. **现金流管理**：手动记账、账单导入、流水查询、预算控制
2. **资产负债管理**：资产账户、负债账户、账户流转
3. **智能化规则**：自动分类、AI 辅助、周期性账单
4. **个性化配置**：财务周期、UI 展示、数据导出

详细功能说明请参考 [核心功能](./2.%20核心功能.md)。

## 开发规划

项目采用分阶段迭代开发：

- **MVP 阶段**：基础记账与资产管理
- **第一阶段**：账单导入与报销管理
- **第二阶段**：数据分析与可视化
- **第三阶段**：预算管理
- **第四阶段**：信贷账户管理
- **第五阶段**：个性化配置
- **第六阶段**：自动化规则与 AI 分类

详细规划请参考 [项目整体开发规划](./3.%20项目整体开发规划.md)。

## 数据模型

系统采用"主表（Transaction）+ 明细表（Entry）"的结构记录财务变动，包含三张核心表：

- `accounts`：账户表（资产、负债、收入、支出、权益）
- `transactions`：交易主表（用于前端展示）
- `entries`：分录明细表（核心底层，记录借贷流向）

所有表主键采用自增整数类型，适配 Dexie.js/IndexedDB。

详细表结构与业务逻辑请参考：
- [核心业务模型](./1.%20核心业务模型.md)
- [各场景流程示例](./90.%20各场景流程示例.md)

## 技术要点

- **数据一致性**：每笔交易必须借贷平衡，自动更新账户余额
- **账单导入**：支持多平台字段映射与重复检测
- **性能优化**：索引优化、增量更新余额

详细实现要点请参考 [技术实现要点](./4.%20技术实现要点.md)。

## 用户体验

- **快速记账**：最少 4 步完成记账，智能默认值
- **视觉反馈**：金额颜色区分、余额变化动画
- **友好提示**：数据校验、重复提醒、预算预警

详细体验设计请参考 [用户体验](./5.%20用户体验.md)。
