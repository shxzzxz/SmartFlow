# 核心功能

本文档列出 SmartFlow 的功能蓝图。每条功能描述系统应如何表达，不直接绑定具体数据 schema；与底层数据模型的对应关系详见《03. 业务领域设计》，与具体表数据示例的对应关系详见《04. 业务场景流程示例》。

> v1 实际交付边界另行讨论；本文档列出的是稳定功能蓝图。

## 1. 现金流管理

### 1.1 手动记账

支持以下业务用途（对应 `business_purpose`）的手工录入：

- 普通支出 / 普通收入
- 普通转账（含手续费）
- 退款（在原支出之下发起）
- 报销垫付 / 报销到账 / 结束报销
- 还款（可含本金、利息、手续费）
- 借入
- 期初余额（账户创建时填写或后续补登）
- 余额调整（理财浮亏浮盈、对账修正）

每个业务用途对应一组固定的业务详情分项（`transaction_details`），label 由系统按场景填充，用户不自由输入。

### 1.2 已入账交易更正

已写入的交易**不直接覆盖原记录**：

- 系统通过"冲销原交易 + 重记正确交易"完成更正
- 原交易 `lifecycle_state` 标记为 `SUPERSEDED`，默认列表中隐藏，详情页可展开历史链路
- 冲销交易使用红字分录（`entries.amount_minor` 为负、保留原借贷方向）

适用范围：金额、账户、分类、备注、时间等任何字段的修改都走同一更正路径。

### 1.3 已入账交易删除

"删除"等价于"冲销且不补记"：

- 写入 `mutation_kind = REVERSAL`、`reversal_reason = DELETE` 的冲销交易
- 原交易 `lifecycle_state` 设为 `VOIDED`
- 默认列表不展示原交易和冲销交易

不允许物理删除已入账交易。

### 1.4 账单导入

- 支持的来源：一木记账、支付宝、微信、国内主流银行的 CSV / Excel 账单
- 流程：上传文件 → 字段映射 → 预览 → 提交
- 预览阶段需展示每条交易的解析结果与命中分类，并可手动调整
- 重复导入检测：以批次指纹 + 行级外部唯一标识结合判定
- 解析过程中若账户 / 分类不存在，按规则映射或落入 `IMPORT_FALLBACK` 系统账户
- 一木记账的解析规则详见《analysis_rules / 一木记账解析规则》

### 1.5 周期性账单

- 用户预设规则（如每月 1 号交房租 ¥3000，记入"居住 / 房租"分类）
- 系统在触发日期自动生成对应交易，`source_kind = AUTO`
- 用户可对自动生成的交易进行更正与删除，遵循 1.2 / 1.3 规则

### 1.6 流水查询

- **流水视图**：按时间倒序排列，默认隐藏 `SUPERSEDED` / `VOIDED` 主记录与 `REVERSAL` 冲销记录
- **筛选维度**：时间范围、账户、分类、business_purpose、关键字
- **子记录折叠**：退款、报销到账等通过 `parent_transaction_id` 挂在父交易下，列表默认折叠到父交易详情
- **详情页**：展示完整的更正 / 冲销链路（包括被隐藏的历史记录）

### 1.7 预算控制

- **总预算**：当月所有支出统一上限
- **分类预算**：绑定到具体支出分类账户（`accounts.account_type = EXPENSE`）
- **超支预警**：达到阈值或超额时高亮、提醒
- **预算趋势**：按日累计已用，支持当月与过往月对比
- **预算豁免**：用户可在 `transaction_details` 上勾选 `is_excluded_from_budget`，整笔交易豁免（细粒度排除以整笔交易为单位）
- **删除 / 迁移分类时**：若分类已有预算引用，预算自动迁移到目标分类；目标分类同月已有预算则金额合并

### 1.8 数据分析

- **结构分析**：饼图展示各支出分类的占比
- **趋势分析**：折线图展示半年 / 一年的收支起伏
- **对比分析**：自动计算同环比（MoM / YoY）

收支聚合走 `transaction_details.stats_kind` + 两个 excluded 字段；分类聚合走 `entries`，按 `account_id` 取借方分录合计。

## 2. 资产负债管理

页面层管理用户视角的"账户"（资产 / 负债 / 权益），底层均落在 `accounts` 单表。

### 2.1 资产账户

- **资金账户**：现金、储蓄卡、第三方支付（`account_subtype = CASH / BANK_CARD / THIRD_PARTY`）
- **理财账户**：基金、股票等（`INVESTMENT`），通过余额调整记录浮亏浮盈
- **报销应收账户**：用于复式表达"公司应付我"（`REIMBURSEMENT`），不参与个人收支统计

### 2.2 负债账户

- **信贷账户**：信用卡、花呗、白条（`CREDIT_CARD`）。需记录账单日（`billing_day`）和还款日（`repayment_day`），支持信用额度（`credit_limit_minor`）
- **债务账户**：房贷、车贷、个人借款（`LOAN` / `CONSUMER_CREDIT`）

### 2.3 权益账户

- 仅由系统按需创建：`OPENING_BALANCE` 等（详见 `system_key`）
- 默认对用户隐藏；专门的"系统账户"视图可查看
- 不允许用户直接创建 EQUITY 账户

### 2.4 账户管理操作

- 创建、编辑、设置图标、备注、排序、隐藏
- **删除**：与分类删除遵循相同的归档机制（详见 3.3）
- 信用卡可设置信用额度、账单日、还款日

### 2.5 账户视图

- **资产 / 负债 / 净资产卡片**：实时聚合
- **账户列表**：按 ASSET / LIABILITY / EQUITY 分组、按 `sort_order` 排序
- **账户流水侧栏**：展开账户的全部 entries 投影回业务交易

## 3. 分类管理

收入 / 支出分类与资产 / 负债账户共享同一张 `accounts` 表，仅 `account_type` 不同。

### 3.1 分类树维护

- 收入与支出分别维护一棵分类树
- 活跃分类最多两级：一级分类、一级 / 二级分类
- 支持新增、编辑、设置图标、隐藏、排序
- 不允许跨类型转移（INCOME 与 EXPENSE 不互通）

### 3.2 删除规则

两条路径：

- **物理删除**：分类无任何 `entries` / `budgets` 引用 → 直接 DELETE
- **归档迁移**：分类存在历史引用 → 标记 `archived_at` 并将 `parent_id` 重指到迁移目标，历史 entries 不变

约束：

- 删除父分类前必须先处理所有活跃子分类，不允许递归级联迁移
- 归档分类不能再作为父节点或新交易的写入目标
- 节点关系最大深度为 3：一级分类 → 二级子分类 → 归档节点

### 3.3 分类与预算 / 报销垫付的联动

- 删除分类时若已有预算引用，预算自动迁移到目标分类
- 报销垫付的支出分类（`transactions.reimbursement_expense_account_id`）即使被归档也仍可被结束报销时少收差额复用

## 4. 智能化规则与自动化

### 4.1 自动分类

- **规则引擎**：用户自定义关键词 / 正则匹配（如包含"麦当劳"自动归"餐饮"）
- **AI 兜底分类**：导入新账单且无规则匹配时调用大语言模型推断
- 用户历史分类可作为 AI 的上下文以提升准确率

### 4.2 周期性账单

详见 1.5。

### 4.3 信贷账户辅助

- 待还金额计算
- 还款日提醒
- 账单日聚合视图

## 5. 偏好与通用设置

### 5.1 财务周期

- 账单月起始日：默认为 1 号，可自定义
- 日历周起始日：默认为周日，可自定义

### 5.2 显示设置

- **分类精简显示**：拥有二级分类的交易在流水页仅显示二级分类名（如只显示"地铁"而非"交通 / 地铁"）
- **显示具体时间**：流水页是否显示到时分秒
- **显示原金额**：发生退款 / 优惠时，视觉上保留原金额并划掉，展示实际金额
- **显示账户图标**：流水列表是否展示资金来源 icon

### 5.3 数据导出

- 格式：Excel / CSV
- 自定义导出字段：时间、金额、分类、账户、备注等
- 时间范围选择：按日期范围导出

### 5.4 其他设置

- 货币符号：默认 ¥
- 小数位数：默认保留 2 位
- 数据备份：完整数据库本地导出（含 `accounts` / `transactions` / `transaction_details` / `entries` / `budgets`）

## 6. 数据安全与隐私

### 6.1 本地优先

- 所有数据存储在设备本地 SQLite
- 不上传任何财务数据到服务器（除非用户主动启用云同步）

### 6.2 备份与恢复

- 手动导出 / 导入完整数据库快照
- 可选云端同步通道（如用户配置 WebDAV / 端到端加密云盘等），v1 不强制

## 7. 未来扩展

以下方向非 v1 重点：

- 多人协作、家庭账本共享、权限管理
- 投资管理：行情同步、收益率计算、资产配置
- 第三方集成：银行 API 自动同步、支付宝 / 微信授权导入
- 智能提醒：异常消费检测、信用卡还款日推送
- iOS 平台支持
