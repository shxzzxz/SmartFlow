# 架构决策

本文档登记 SmartFlow 在演进过程中做出的、影响跨模块协作或长期演进方向的关键决策。

每条决策只在本文写一条**摘要**（约束 + 一句话理由），完整背景、含义与反例放到对应的 `08.x` 子文档。新增条目时优先沿用已有编号顺序，不重排历史编号。

## 何时新增条目

- 决策**约束的是跨层 / 跨模块边界**（例如：A 层不得依赖 B 层、B 模块通过 X 协议接入核心层）
- 决策一旦反悔会引起**广泛重构**，未来贡献者需要先理解再扩展
- 仅影响单文件 / 单模块的局部权衡**不**进本文档，写到代码注释或对应模块的 README 即可

## 子文档模板

每个 `08.x` 子文档建议采用以下结构，便于横向比较：

- **背景**：触发决策的问题、之前的状态
- **决策**：用一句陈述句概括做什么 / 不做什么
- **含义**：对代码组织、调用方向、UI 路径的影响
- **适用案例**：决策落地的具体场景
- **反例**：违反本决策的常见诱惑性方案，以及为什么不取
- **演进规则**：未来出现相似场景时如何沿用
