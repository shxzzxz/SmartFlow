# 核心业务规则

本文承接《03. 业务领域设计》的核心业务规则章节，完整描述交易写入、更正、删除、业务场景与统计口径。

## 1. 核心业务规则

### 1.1 一笔交易的完整表达

每一笔交易由三部分在同一数据库事务内提交：

- 1 条 `transactions` 主记录
- 1 条或多条 `transaction_details` 分项
- 2 条或多条 `entries` 分录

三者通过 `transaction_id` 关联，但 `transaction_details` 与 `entries` 不互相引用。

写入约束：

- entries 借贷必须平衡
- transaction_details 至少 1 条
- transaction_details 的 `amount_minor` 总和与 entries 的资金合计无强约束 —— details 是业务分项信息，entries 是账务真相
- 统计排除与预算排除是整笔交易口径，记录在 `transactions.is_excluded_from_stats` / `transactions.is_excluded_from_budget`

### 1.2 更正机制

已写入的交易不直接更新原记录，更正分两步：

1. 生成一条 `mutation_kind = REVERSAL` 的冲销交易，`mutation_previous_transaction_id` 指向被更正的 CURRENT 交易，`mutation_reason = CORRECTION`
2. 生成一条 `mutation_kind = CORRECTION` 的新交易，`mutation_previous_transaction_id` 指向上一步生成的 REVERSAL 交易，承载正确金额 / 账户 / 分类

原交易 `business_state` 设为 `REPLACED`，冲销交易 `business_state = COMPENSATION`，更正后的新交易 `business_state = CURRENT`。

多次更正时，每次都从当前 `business_state = CURRENT` 的链尾继续追加：

```text
ORIGINAL(1) -> REVERSAL(2) -> CORRECTION(3) -> REVERSAL(4) -> CORRECTION(5)
```

其中 `1 / 3 / 5` 是业务版本记录，`2 / 4` 是对应的账务冲销补偿记录。

### 1.3 红字冲销

冲销交易在 entries 上保留原借贷方向，`amount_minor` 取负数。

举例：原 `account=餐饮支出, direction=DEBIT, amount=100`，冲销分录写入 `account=餐饮支出, direction=DEBIT, amount=-100`。

效果：

- 账户余额抵销
- 借贷方向不翻转，便于审计和追溯
- transactions 头部金额仍记原始正值

### 1.4 删除机制

已写入交易不允许物理删除。"删除"统一走冲销路径：

- 生成一条 `mutation_kind = REVERSAL`、`mutation_reason = DELETE` 的冲销交易
- 冲销交易 `mutation_previous_transaction_id` 指向被删除的 CURRENT 交易
- 原交易 `business_state` 设为 `CANCELED`
- 冲销交易 `business_state` 设为 `COMPENSATION`
- 默认列表不展示原交易和冲销交易

`mutation_reason` 用于区分冲销动机：

- `CORRECTION` —— 用户在更正流程中产生的冲销
- `DELETE` —— 用户在删除流程中产生的冲销，且没有后续 CORRECTION

### 1.5 业务场景规则

本节描述每种业务场景的"应该如何表达"。具体表数据示例见《03.3 业务场景流程示例》。

#### 1.5.1 普通支出

- `business_purpose = DAILY_EXPENSE`
- detail：1 条 `PRIMARY_EXPENSE`
- entries：支出分类账户 DEBIT，资金账户 CREDIT

#### 1.5.2 普通收入

- `business_purpose = DAILY_INCOME`
- detail：1 条 `PRIMARY_INCOME`
- entries：资金账户 DEBIT，收入分类账户 CREDIT

#### 1.5.3 普通转账

- `business_purpose = TRANSFER`
- detail：1 条 `TRANSFER_MAIN`；如有手续费再加 1 条 `TRANSFER_FEE`
- entries：
  - 无手续费：转入资产账户 DEBIT 主体转账金额，转出资产账户 CREDIT 主体转账金额
  - 有手续费：转入资产账户 DEBIT 主体转账金额，手续费支出分类 DEBIT 手续费，转出资产账户 CREDIT 主体转账金额 + 手续费

#### 1.5.4 退款

- `business_purpose = REFUND`
- `root_transaction_id` 与原支出主记录相同
- `parent_transaction_id` 指向父业务根交易
- detail：1 条 `REFUND_MAIN`
- entries：资金账户 DEBIT，原支出分类账户 CREDIT（红字效果由账户语义自然达成，不必使用红字写法）

#### 1.5.5 报销垫付

- `business_purpose = REIMBURSEMENT_ADVANCE`
- `reimbursement_expense_account_id` 记录用户为本次报销选择的支出分类
- detail：1 条 `REIMBURSEMENT_ADVANCE_MAIN`
- entries：报销应收账户（ASSET / REIMBURSEMENT 子类型）DEBIT，资金或负债账户 CREDIT

#### 1.5.6 报销到账（部分到账）

- `business_purpose = REIMBURSEMENT_RECEIPT`
- `root_transaction_id` 与报销垫付主记录相同
- `parent_transaction_id` 指向报销垫付业务根交易
- detail：1 条 `REIMBURSEMENT_RECEIPT_MAIN`
- entries：资金账户 DEBIT，报销应收账户 CREDIT

#### 1.5.7 结束报销（含差额）

结束报销 = 剩余全额报销 + 差额收支。

- `business_purpose = REIMBURSEMENT_CLOSE`
- `root_transaction_id` 与报销垫付主记录相同
- `parent_transaction_id` 指向报销垫付业务根交易
- detail 组合：
  - 1 条 `REIMBURSEMENT_CLOSE_MAIN`，金额为剩余应收金额
  - 若多收：再加 1 条 `REIMBURSEMENT_GAP_INCOME`
  - 若少收：再加 1 条 `REIMBURSEMENT_GAP_EXPENSE`
- entries：
  - 资金账户 DEBIT 实收金额
  - 报销应收账户 CREDIT 剩余应收金额
  - 多收差额：报销差额收入分类（懒加载 `system_key = REIMBURSEMENT_GAP_INCOME`）CREDIT 差额
  - 少收差额：原报销支出分类（取自 `transactions.reimbursement_expense_account_id`）DEBIT 差额

#### 1.5.8 还款（含本息、优惠）

- `business_purpose = DEBT_REPAYMENT`
- detail 组合：
  - 1 条 `REPAYMENT_PRINCIPAL`
  - 若有利息：1 条 `REPAYMENT_INTEREST`
  - 若有手续费：1 条 `REPAYMENT_FEE`
  - 若有优惠：1 条 `REPAYMENT_DISCOUNT`
- 实际支付金额 = 本金 + 利息 + 手续费 - 优惠；`primary_amount_minor` 取实际支付金额。
- entries：
  - 资金账户 CREDIT 实际支付金额
  - 负债账户 DEBIT 本金
  - 利息支出分类 DEBIT 利息
  - 手续费支出分类 DEBIT 手续费
  - 优惠收入分类 CREDIT 优惠金额

#### 1.5.9 借入

- `business_purpose = BORROWING`
- detail：1 条 `BORROWING_PRINCIPAL`
- entries：资金账户 DEBIT，负债账户 CREDIT
- 债务期初导入（无对方资产）：资金侧由系统期初余额账户对冲

#### 1.5.10 期初余额

- `business_purpose = OPENING_BALANCE`
- detail：1 条 `OPENING_BALANCE_MAIN`
- entries：目标账户按其类型方向（资产 DEBIT / 负债 CREDIT）；对手方为系统期初余额权益账户（懒加载 `system_key = OPENING_BALANCE`）

#### 1.5.11 余额调整

- `business_purpose = BALANCE_ADJUSTMENT`
- detail：1 条 `BALANCE_ADJUSTMENT_MAIN`
- entries：目标账户按差额方向调整，对手方为系统期初余额权益账户

### 1.6 统计规则

#### 1.6.1 账户维度

账户当前余额：从 entries 按 `account_id` 累计借贷净额，写入分录时同步更新 `accounts.balance_minor`。

账户流水：从 entries 反查，按 `transactions.occurred_at` 倒序；默认只展示 `business_state = CURRENT` 的 transactions

#### 1.6.2 流水维度

默认流水列表：

- 来源 `transactions`，按 `root_transaction_id` 聚合业务流水，按当前有效主记录的 `occurred_at` 倒序
- 过滤条件：`business_state = CURRENT`
- 当前有效主记录：同一 `root_transaction_id` 下，`parent_transaction_id IS NULL` 且 `business_state = CURRENT`
- 当前有效子记录：同一 `root_transaction_id` 下，`parent_transaction_id IS NOT NULL` 且 `business_state = CURRENT`
- 子记录（退款、报销到账等）默认折叠到父交易详情中

筛选维度：时间范围、账户、分类、business_purpose、关键字。

#### 1.6.3 收支统计维度

**收支统计的唯一真相在 `entries`，通过分类账户与借贷方向计算。`transaction_details` 不参与统计口径。**

月度总收入：

```text
SUM(
  CASE
    WHEN entries.direction = 'CREDIT' THEN entries.amount_minor
    WHEN entries.direction = 'DEBIT' THEN -entries.amount_minor
  END
) FROM entries
JOIN transactions ON entries.transaction_id = transactions.id
JOIN accounts ON entries.account_id = accounts.id
WHERE accounts.account_type = 'INCOME'
  AND transactions.business_state = 'CURRENT'
  AND transactions.is_excluded_from_stats = false
```

月度总支出：

```text
SUM(
  CASE
    WHEN entries.direction = 'DEBIT' THEN entries.amount_minor
    WHEN entries.direction = 'CREDIT' THEN -entries.amount_minor
  END
) FROM entries
JOIN transactions ON entries.transaction_id = transactions.id
JOIN accounts ON entries.account_id = accounts.id
WHERE accounts.account_type = 'EXPENSE'
  AND transactions.business_state = 'CURRENT'
  AND transactions.is_excluded_from_stats = false
```

退款等抵减支出的场景会在 EXPENSE 账户上产生 CREDIT 分录，因此自然以负支出抵减统计；收入冲销同理。

#### 1.6.4 分类维度

按支出分类聚合 → 走 entries，并按账户类型与借贷方向计算统计增量：

```text
SUM(
  CASE
    WHEN entries.direction = 'DEBIT' THEN entries.amount_minor
    WHEN entries.direction = 'CREDIT' THEN -entries.amount_minor
  END
) FROM entries
JOIN transactions ON entries.transaction_id = transactions.id
JOIN accounts ON entries.account_id = accounts.id
WHERE accounts.id IN (目标分类及其归档子节点)
  AND accounts.account_type = 'EXPENSE'
  AND transactions.business_state = 'CURRENT'
  AND transactions.is_excluded_from_stats = false
```

归档分类的历史 entries 自动归集到迁移目标，因为归档时 `parent_id` 已经重指。

收入分类聚合同理：`INCOME` 账户 CREDIT 记正收入，DEBIT 记负收入。

#### 1.6.5 预算维度

预算聚合走支出分类 entries，并叠加 `transactions.is_excluded_from_budget` 过滤：

```text
某分类某月已用预算 =
  SUM(
    CASE
      WHEN direction = DEBIT THEN amount_minor
      WHEN direction = CREDIT THEN -amount_minor
    END
  )
  WHERE 命中支出分类树
    AND transactions.business_state = CURRENT
    AND transactions.is_excluded_from_budget = false
```

预算排除是**整笔交易粒度**：一笔交易只要 `is_excluded_from_budget = true`，其所有支出分类 entries 均不计入预算。
