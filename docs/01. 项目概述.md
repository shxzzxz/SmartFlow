# 项目概述

## 项目定位

SmartFlow 是一款基于**复式记账法**的个人财务管理 Android 应用。所有数据存储在设备本地，强调隐私、可控、可审计。

## 核心特点

- **会计原则驱动**：严格遵循复式记账法，确保每一笔交易借贷平衡、可追溯（详见《03. 业务领域设计》）
- **本地优先**：数据保存在设备本地数据库
- **智能化辅助**：账单导入、规则与 AI 分类、周期性账单等自动化能力
- **多维度分析**：分类结构、收支趋势、同环比对比

## 核心模块

1. **现金流管理**：手动记账、账单导入、流水查询、预算控制
2. **资产负债管理**：资产账户、负债账户、账户流转
3. **分类管理**：收入 / 支出分类树
4. **智能化规则**：自动分类、AI 辅助、周期性账单
5. **个性化配置**：财务周期、UI 展示、数据导出

详细功能说明请参考《02. 核心功能》。

## 技术栈

- **应用框架**：Flutter（初期仅 Android）
- **本地数据库**：drift（SQLite）
- **状态管理**：Riverpod 3.x + Code Generation
- **路由**：go_router
- **金额**：decimal + 自封装 Money 类（按"分"作为整数存储）
- **日期 / 国际化**：DateTime + intl + flutter_localizations
- **代码生成**：freezed + json_serializable + drift + riverpod_generator + build_runner
- **图表**：fl_chart
- **UI**：Material 3 + 自定义 ThemeData token
- **测试**：flutter_test + mocktail
- **代码质量**：flutter_lints

技术分层与目录组织详见《06. 项目结构说明》。视觉规范详见《05. 设计系统》。

## 业务领域设计：数据模型与业务规则

- `accounts`：单表树形结构，统一承载五大类账户（资产 / 负债 / 权益 / 收入 / 支出）。收入与支出分类作为账户树的特殊子树存在。归档分类通过 `archived_at` 与 `parent_id` 重指实现历史归集
- `transactions`：交易主记录，承载业务用途、业务关系、业务状态、更正链路与统计 / 预算排除标记
- `transaction_details`：业务详情分项，承载分项类型（`detail_type`）与分项金额，用于还款、报销等复合交易的结构表达
- `entries`：复式记账分录，承载借贷方向、金额与影响账户；账户余额、收支统计、分类统计与预算统计均从 entries 推导
- `budgets`：月度预算，绑定支出分类账户
