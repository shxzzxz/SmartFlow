# 数据模型

本文承接《03. 业务领域设计》的核心数据模型章节，完整描述 SmartFlow 的核心表、字段语义与关键约束。

## 1. 核心数据模型

本节列出表与关键字段；详细字段类型、索引与迁移在 drift schema 中定义。

- 所有金额字段以最小货币单位（如人民币的"分"）作为整数存储。
- 每个表均包含基础字段 created_at、updated_at

### 1.1 accounts

承载五大类账户与收入 / 支出分类的单表树形结构。

```text
accounts
- id
- name
- account_type           ASSET / LIABILITY / EQUITY / INCOME / EXPENSE
- account_subtype        资产细分（CASH / BANK_CARD / THIRD_PARTY / INVESTMENT / REIMBURSEMENT）
                         负债细分（CREDIT_CARD / LOAN / CONSUMER_CREDIT）
                         其他类型为 NULL
- parent_id              树形父节点；顶级账户 / 顶级分类为 NULL
- currency_code          币种
- balance_minor          当前余额，由 entries 实时累计
- icon_key
- note
- credit_limit_minor     仅信贷账户使用
- billing_day            仅信贷账户使用
- repayment_day          仅信贷账户使用
- sort_order
- is_hidden              用户主动隐藏（仍可记账）
- archived_at            归档时间，非空表示该节点不再作为活跃写入目标
- system_key             系统语义身份，见下文
- source                 创建来源：builtin / user / import
```

**`system_key` 取值**：

- `openingBalance` —— 期初余额对冲账户（EQUITY，按币种唯一）
- `reimbursementGapIncome` —— 报销差额收入分类
- `debtInterestExpense` —— 借贷利息支出分类
- `debtFeeExpense` —— 借贷手续费支出分类
- `lendingExpense` —— 借出分类
- `borrowingIncome` —— 借入分类
- `importFallback` —— 导入兜底账户

约束：

- `system_key + currency_code` 唯一
- `system_key` 非空的节点不允许物理删除，可改名 / 改图标 / 改排序 / 隐藏
- `source` 只表达创建来源，不决定是否可删除；删除保护由 `system_key` 是否非空决定

**层级与归档规则**：

- INCOME / EXPENSE 分类的活跃节点最多两级：一级分类、一级 / 二级分类
- 归档节点（`archived_at IS NOT NULL`）不能再作为父节点，也不能再作为新交易的写入目标
- 节点关系最大深度为 3：一级分类 → 二级子分类 → 归档节点
- 删除分类的两条路径：
  - 无任何 entries / 预算引用 → 物理删除
  - 存在历史引用 → 标记 `archived_at` 并将 `parent_id` 重指到迁移目标，历史 entries 不变。其归档子节点将 `parent_id` 重指到相同迁移目标
- 删除父分类前必须先处理所有活跃子分类，不允许递归级联迁移

### 1.2 transactions

业务事件主记录。

```text
transactions
- id
- root_transaction_id               所属业务聚合根交易 id，见下文语义
- business_purpose                  业务用途，见下文枚举
- occurred_at                       交易发生时间
- currency_code
- primary_amount_minor              交易事件金额；用于头部展示、搜索与导入匹配，账务真相以 entries 为准
- counterparty_name                 对手方名称
- note                              用户备注
- parent_transaction_id             子记录所属父业务交易 id
- reimbursement_expense_account_id  仅报销垫付主链使用，记录原报销支出分类
                                    用于结束报销时少收差额复用同一分类
- mutation_kind                     ORIGINAL / CORRECTION / REVERSAL
- mutation_previous_transaction_id  更正链中的上一条交易记录
- mutation_reason                   CORRECTION / DELETE
- business_state                    CURRENT / REPLACED / CANCELED / COMPENSATION
- is_excluded_from_stats            默认 false；整笔交易不计入收支统计
- is_excluded_from_budget           默认 false；整笔交易不计入预算统计
- source_kind                       MANUAL / IMPORT / AUTO
```

**业务聚合字段语义**：

- `root_transaction_id` 表示所属业务聚合根交易 id。
- 独立主记录的 `root_transaction_id` 指向自身 id；
- 主记录的更正 / 冲销、子记录、子记录的更正 / 冲销，均使用同一个 `root_transaction_id`，指向最初父记录。

**`business_purpose` 枚举**：

```text
DAILY_EXPENSE             普通支出
DAILY_INCOME              普通收入
TRANSFER                  普通转账
REFUND                    退款
REIMBURSEMENT_ADVANCE     报销垫付
REIMBURSEMENT_RECEIPT     报销到账（部分到账）
REIMBURSEMENT_CLOSE       结束报销（最后一笔，可能含差额）
DEBT_REPAYMENT            还款
BORROWING                 借入
OPENING_BALANCE           期初余额
BALANCE_ADJUSTMENT        余额调整
```

冲销交易不单独使用枚举值，而是继承被冲销交易的 `business_purpose`，通过 `mutation_kind = REVERSAL` 区分。

**更正链字段语义**：

- `ORIGINAL`：`mutation_previous_transaction_id = NULL`
- `REVERSAL`：`mutation_previous_transaction_id` 指向被冲销的当前有效记录
- `CORRECTION`：`mutation_previous_transaction_id` 指向同一次更正流程生成的 `REVERSAL` 记录

更正链按实际追加事件组织为单链表。例如：`原记录 -> 冲销1 -> 新交易1 -> 冲销2 -> 新交易2`。

**业务状态语义**：

- `CURRENT` —— 当前有效业务记录
- `REPLACED` —— 已被更正替代的历史记录
- `CANCELED` —— 已被删除路径取消的历史记录
- `COMPENSATION` —— 冲销补偿记录

`business_state` 只表达用户业务视角下的有效性；`mutation_kind` 表达更正链角色。二者共同描述一条交易记录，但不互相替代。

### 1.3 transaction_details

业务详情分项。承载复合交易的业务组成信息

```text
transaction_details
- id
- transaction_id
- line_no                       同一交易内排序
- detail_type                   分项类型，受控枚举（见下）
- amount_minor                  分项金额
```

**`detail_type` 受控枚举**（按 `business_purpose` 分组）：

| business_purpose        | 允许的 detail_type                                                                       |
|:------------------------|:--------------------------------------------------------------------------------------|
| `DAILY_EXPENSE`         | `PRIMARY_EXPENSE`                                                                     |
| `DAILY_INCOME`          | `PRIMARY_INCOME`                                                                      |
| `TRANSFER`              | `TRANSFER_MAIN` / `TRANSFER_FEE`                                                      |
| `REFUND`                | `REFUND_MAIN`                                                                         |
| `REIMBURSEMENT_ADVANCE` | `REIMBURSEMENT_ADVANCE_MAIN`                                                          |
| `REIMBURSEMENT_RECEIPT` | `REIMBURSEMENT_RECEIPT_MAIN`                                                          |
| `REIMBURSEMENT_CLOSE`   | `REIMBURSEMENT_CLOSE_MAIN` / `REIMBURSEMENT_GAP_EXPENSE` / `REIMBURSEMENT_GAP_INCOME` |
| `DEBT_REPAYMENT`        | `REPAYMENT_PRINCIPAL` / `REPAYMENT_INTEREST` / `REPAYMENT_FEE`                        |
| `BORROWING`             | `BORROWING_PRINCIPAL`                                                                 |
| `OPENING_BALANCE`       | `OPENING_BALANCE_MAIN`                                                                |
| `BALANCE_ADJUSTMENT`    | `BALANCE_ADJUSTMENT_MAIN`                                                             |


**与 entries 的关系**：

- `transaction_details` 与 `entries` **不互相外键引用**
- details 说明业务分项结构；entries 表达复式入账、账户余额、收支统计、预算统计
- 同一笔交易，detail 数与 entry 数无对应关系
- details 不参与借贷平衡，不决定收入 / 支出 / 预算口径

### 1.4 entries

复式记账分录。

```text
entries
- id
- transaction_id
- account_id
- direction              DEBIT / CREDIT
- amount_minor           允许负数，仅用于红字冲销
```

约束：

- 同一 `transaction_id` 下，借方 `amount_minor` 合计 = 贷方 `amount_minor` 合计
- 红字冲销保留原借贷方向，`amount_minor` 取负数
- 写入后不允许 update / delete，更正与删除通过新交易冲销实现
- `account_id` 指向归档账户（archived）的 entries 仍合法存在，仅作为历史归集；新交易不得写入归档账户

### 1.5 budgets

月度预算。

```text
budgets
- id
- month_key             形如 202605，避免日期时区问题
- account_id            支出分类账户 id；NULL 表示当月总预算
- amount_minor
- currency_code
```

约束：

- `account_id` 非空时只能指向 `account_type = EXPENSE` 的活跃账户
- `(month_key, account_id, currency_code)` 唯一；`account_id` 为 NULL 时按 `(month_key, currency_code)` 唯一
- 删除 / 迁移分类时，若该分类已有预算引用：预算迁移到目标分类；若目标分类同月已有预算，金额合并

### 1.6 app_metadata

应用级元数据，用于记录非结构类数据版本，例如内置数据版本。

```text
app_metadata
- key
- value
- updated_at
```

当前使用：

- `builtin_data_version` —— 内置数据版本，用于后续升级补齐新增系统内置数据
