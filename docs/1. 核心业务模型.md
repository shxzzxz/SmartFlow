# 核心业务模型

SmartFlow 系统底层采用**复式记账法（Double-Entry Bookkeeping）**，以确保资金流转、报销垫付、债务还款等复杂财务场景下的数据一致性与准确性。

## 1. 核心会计等式

系统的数据流转严格遵循以下等式推导：

* **基础等式**：`资产 = 负债 + 所有者权益（净资产）`
* **动态等式**：`资产 = 负债 + 所有者权益 + (收入 - 支出)`
* **记账操作等式**：`资产 + 支出 = 负债 + 所有者权益 + 收入`（该等式下，所有变量均为正数，是系统分录的基础）。

## 2. 五大类账户说明 (Account Types)

在复式记账模型中，日常的“收支分类”与“资金载体”统一抽象为“账户”。系统共包含五大类账户：

| 账户类型 | 英文标识 | 业务含义与示例 |
| :--- | :--- | :--- |
| **资产** | ASSET | 资金的实际载体。如：储蓄卡、现金、支付宝、基金、**应收账款（报销款）**。 |
| **负债** | LIABILITY | 欠款或透支额度。如：信用卡、花呗、房贷、车贷。 |
| **所有者权益** | EQUITY | 净资产的调节项。如：初始余额设定、理财产品的“平账/浮亏浮盈”。 |
| **收入** | INCOME | 资金流入的分类。如：工资、兼职收入、投资分红。 |
| **支出** | EXPENSE | 资金流出的分类。如：餐饮、交通、房租。 |

## 3. 借贷记账规则 (Debit & Credit)

系统要求：**每一笔交易（Transaction）至少涉及两个账户，且借方（Debit）总额必须等于贷方（Credit）总额。**

在代码逻辑中，借贷代表资金的流转方向，具体映射规则如下：

* **资产 (ASSET) / 支出 (EXPENSE)**：
  * 账户金额**增加**，记为 **借 (DEBIT)**
  * 账户金额**减少**，记为 **贷 (CREDIT)**
* **负债 (LIABILITY) / 收入 (INCOME) / 所有者权益 (EQUITY)**：
  * 账户金额**增加**，记为 **贷 (CREDIT)**
  * 账户金额**减少**，记为 **借 (DEBIT)**

## 4. 核心数据模型与表结构 (MySQL 形式)

系统采用“主表（Transaction）+ 明细表（Entry）”的结构记录财务变动。

### 4.1 账户表 `accounts`

管理所有资产负债实体及收支分类。

```sql
CREATE TABLE accounts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) NOT NULL,          -- 账户名称，如"招商银行", "餐饮", "工资"
    type VARCHAR(20) NOT NULL,          -- 类型：ASSET, LIABILITY, EQUITY, INCOME, EXPENSE
    parent_id INTEGER NULL,             -- 父账户ID（用于构建多级分类/子账户）
    currency VARCHAR(10) DEFAULT 'CNY', -- 币种
    balance DECIMAL(15, 2) DEFAULT 0.00,-- 账户当前余额（通过凭证触发器或事务同步更新）
    icon_url VARCHAR(255),              -- UI图标
    billing_day INT NULL,               -- 账单日（仅LIABILITY生效）
    repayment_day INT NULL,             -- 还款日（仅LIABILITY生效）
    is_hidden BOOLEAN DEFAULT FALSE,    -- 是否隐藏
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 交易主表 `transactions`

用于前端列表展示、时间轴聚合及业务类型归类。

```sql
CREATE TABLE transactions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    tx_type VARCHAR(20) NOT NULL,       -- 业务类型：EXPENSE(支出), INCOME(收入), TRANSFER(流转), REFUND(退款), ADJUSTMENT(平账)
    amount DECIMAL(15, 2) NOT NULL,     -- 交易绝对值金额（用于UI展示）
    original_amount DECIMAL(15, 2) NULL,-- 原金额（用于展示退款划线价）
    tx_time DATETIME NOT NULL,          -- 交易发生时间
    payee VARCHAR(100) NULL,            -- 交易对手方
    description VARCHAR(255) NULL,      -- 备注
    parent_tx_id INTEGER NULL,          -- 关联原交易ID（用于退款关联）
    source VARCHAR(50) DEFAULT 'MANUAL',-- 数据来源：MANUAL, IMPORT, AUTO
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 分录明细表 `entries` (核心底层)

记录资金的具体流向。

```sql
CREATE TABLE entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    transaction_id INTEGER NOT NULL,
    account_id INTEGER NOT NULL,
    flow_direction VARCHAR(10) NOT NULL, -- 借贷方向：DEBIT 或 CREDIT
    amount DECIMAL(15, 2) NOT NULL,      -- 单条分录的绝对值金额
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.4 余额更新逻辑说明 (System Constraint)

每次向 `entries` 插入数据时，需同步更新 `accounts` 表的 `balance` 字段。更新算法如下：

1. 校验 `account_type`。
2. 若为 `ASSET` 或 `EXPENSE`：遇 `DEBIT` 执行 `+amount`，遇 `CREDIT` 执行 `-amount`。
3. 若为 `LIABILITY`, `INCOME` 或 `EQUITY`：遇 `CREDIT` 执行 `+amount`，遇 `DEBIT` 执行 `-amount`。
